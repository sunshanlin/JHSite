ดึงอัตราแลกเปลี่ยนและวันหยุด ธปท. เข้า BC อัตโนมัติ

การคีย์อัตราแลกเปลี่ยนและวันหยุดด้วยมือดูเป็นงานเล็ก แต่ข้อมูลที่ผิดหรือขาดอาจกระทบมูลค่ารายการ กำไรขาดทุนจากอัตราแลกเปลี่ยน และวันที่ใช้วางแผนรับจ่ายเงิน

โดย Jirapat Wichayapong · ที่ปรึกษาและนักพัฒนา Business Central

อัตราแลกเปลี่ยนถูกใช้มากกว่าหนึ่งจุด

ธุรกรรมเงินตราต่างประเทศต้องใช้อัตราแลกเปลี่ยนในหลายช่วง โดยวันที่และชนิดเรตต้องเป็นไปตามนโยบายบัญชีและภาษีของกิจการ

  • ตอนบันทึกรายการ — ใช้แปลงมูลค่าเป็นสกุลเงินท้องถิ่น
  • ตอนรับหรือจ่ายเงิน — ใช้คำนวณผลต่างจากอัตราเดิมที่บันทึกไว้
  • วันสิ้นงวด — ใช้ปรับมูลค่ายอดคงเหลือที่เป็นเงินตราต่างประเทศ

Business Central เก็บอัตราแลกเปลี่ยนพร้อมวันที่เริ่มมีผล และใช้อัตราที่มีผลกับวันที่ของเอกสาร หากข้อมูลวันใหม่ยังไม่ถูกบันทึก เอกสารอาจยังใช้อัตราก่อนหน้าได้ จึงควรมีการตรวจสอบว่าข้อมูลของวันที่ต้องการเข้าระบบแล้วจริง

อย่าเลือกเรตจากชื่ออย่างเดียว

BOT API มีข้อมูลหลายชุด เช่น อัตราซื้อ อัตราขาย และอัตราเฉลี่ย ต้องกำหนดให้ชัดว่าแต่ละธุรกรรมจะใช้ชุดใด ไม่ควรดึงตัวเลขแรกที่ API ส่งกลับมาใช้กับทุกกรณี

เลือกแหล่งข้อมูลและกติกาให้ชัด

ธนาคารแห่งประเทศไทยเผยแพร่ข้อมูลอัตราแลกเปลี่ยนผ่าน BOT API ซึ่งระบบสามารถเชื่อมต่อและนำเข้าตามรอบเวลาได้ ข้อดีคือ

  • ตรวจสอบที่มาได้ — รู้ว่าเป็นข้อมูลชุดใดและอ้างอิงวันที่ใด
  • ใช้กติกาเดียวกัน — ลดความต่างระหว่างข้อมูลที่แต่ละฝ่ายเลือกใช้
  • ลดงานประจำ — ไม่ต้องคีย์ข้อมูลเดิมทุกวันทำการ

ก่อนเชื่อมต่อ ต้องอนุมัติผังการใช้งานว่าธุรกรรมใดใช้เรตประเภทใด วันที่ใด และปัดเศษอย่างไร เพราะเรตซื้อ เรตขาย และเรตเฉลี่ยให้ผลต่างกัน

วันหยุดต้องมาพร้อมกติกาการเลื่อนวัน

BOT API ยังเผยแพร่วันหยุดตามประเพณีของสถาบันการเงิน แต่การมีรายการวันหยุดอย่างเดียวยังไม่พอ ระบบต้องรู้ด้วยว่าแต่ละกระบวนการจะเลื่อนวันอย่างไร

  • วันครบกำหนดอาจเลื่อนไปวันทำการถัดไปหรือย้อนมาก่อนวันหยุด ขึ้นกับข้อตกลง
  • วันวางบิลมักต้องเลื่อนมาก่อน เพื่อไม่ให้พลาดรอบของลูกค้า
  • วันที่คาดว่าจะรับหรือจ่ายเงินควรอ้างอิงปฏิทินเดียวกับกระบวนการจริง

Base Calendar มาตรฐานของ Business Central รองรับวันทำงานและวันหยุดสำหรับการคำนวณวันที่ด้านโลจิสติกส์ ส่วนการนำวันหยุดไปคำนวณวันวางบิลหรือวันรับจ่ายอาจต้องเพิ่มกติกาเฉพาะขององค์กร

สิ่งที่ควรตั้งให้ระบบทำเอง

  1. ดึงข้อมูลตามกำหนดเวลา พร้อมบันทึกชื่อชุดข้อมูลและวันที่อ้างอิง
  2. แปลงและบันทึกอัตราให้ตรงกับโครงสร้าง Currency Exchange Rates ของ BC
  3. อัปเดตวันหยุด แล้วใช้กติกาเลื่อนวันที่ที่กำหนดไว้สำหรับแต่ละงาน
  4. แจ้งเตือนเมื่อดึงไม่สำเร็จหรือไม่พบข้อมูลของวันที่คาดไว้

ข้อควรพิจารณาก่อนเชื่อมต่อ

การเชื่อมข้อมูลภายนอกเข้าระบบบัญชีควรมีการควบคุมอย่างน้อยดังนี้

  • รองรับการดึงย้อนหลัง เมื่อ API หรือ Job Queue ทำงานไม่สำเร็จ
  • ไม่ทับข้อมูลโดยไม่ตั้งใจ โดยเฉพาะอัตราที่บัญชีอนุมัติให้ใช้เฉพาะรายการ
  • เก็บประวัติการทำงาน ทั้งเวลาที่ดึง สถานะ และข้อความผิดพลาด

JWIC Localization ช่วยตรงไหน

ส่วนเสริมที่เราพัฒนารองรับการดึงอัตราแลกเปลี่ยนและวันหยุดจาก BOT API เข้าระบบอัตโนมัติ พร้อมวันที่อ้างอิง บันทึกผลการทำงาน และการแจ้งเตือนเมื่อเกิดข้อผิดพลาด

การตั้งค่าชนิดเรตและกติกาเลื่อนวันจะแยกตามนโยบายของแต่ละองค์กร เพื่อไม่ให้การเชื่อม API เปลี่ยนหลักบัญชีหรือเงื่อนไขทางธุรกิจโดยไม่ตั้งใจ

อ้างอิง: BOT API, การอัปเดตอัตราแลกเปลี่ยนใน Business Central และ การตั้งค่า Base Calendar
