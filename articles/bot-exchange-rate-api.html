BOT API / Data Service

รู้จักกับบริการข้อมูลจาก ธปท. อัตราแลกเปลี่ยนและวันหยุดธนาคาร

ธนาคารแห่งประเทศไทยเปิดบริการ BOT API ให้ระบบภายนอกเรียกใช้ข้อมูลได้โดยตรง บทความนี้พาไปรู้จักสองชุดข้อมูลที่ใช้ในงานธุรกิจบ่อย ได้แก่ อัตราแลกเปลี่ยนและวันหยุดของสถาบันการเงิน รวมถึงข้อควรรู้ก่อนนำไปใช้กับ Business Central

โดย Jirapat Wichayapong · ที่ปรึกษาและนักพัฒนา Business Central

BOT API คืออะไร

BOT API คือช่องทางที่ ธปท. ใช้เผยแพร่ข้อมูลให้เว็บไซต์ แอปพลิเคชัน และระบบงานเรียกใช้แบบอัตโนมัติ แทนการเปิดหน้าเว็บแล้วคัดลอกข้อมูลไปกรอกเอง

ข้อมูลที่ให้บริการมีหลายกลุ่ม ตั้งแต่อัตราแลกเปลี่ยน อัตราดอกเบี้ย สถิติทางเศรษฐกิจ ไปจนถึงวันหยุดของสถาบันการเงิน สำหรับการเชื่อมต่อ ระบบต้องสมัครใช้บริการและส่ง API key ผ่านส่วนหัว Authorization ตามคู่มือของแต่ละ API

ข้อมูลจากต้นทาง ไม่ได้แปลว่าใช้ได้ทันทีทุกกรณี

API ช่วยให้ได้ข้อมูลอย่างสม่ำเสมอ แต่ผู้ใช้งานยังต้องกำหนดว่าจะเลือกข้อมูลชุดใด ใช้กับงานอะไร และจัดการอย่างไรเมื่อข้อมูลของวันที่ต้องการยังไม่ประกาศ

บริการข้อมูลอัตราแลกเปลี่ยน

ผลิตภัณฑ์ Exchange Rates ของ ธปท. มีข้อมูลเงินตราต่างประเทศเทียบกับเงินบาท โดยบริการหลักครอบคลุม

  • อัตราแลกเปลี่ยนถัวเฉลี่ยของธนาคารพาณิชย์ — เงินตราต่างประเทศหลายสกุลเทียบกับเงินบาท
  • อัตราอ้างอิงระหว่างธนาคาร — อัตราถัวเฉลี่ยถ่วงน้ำหนักระหว่างธนาคารสำหรับเงินบาทต่อดอลลาร์สหรัฐ
  • หลายช่วงเวลา — เรียกดูได้ทั้งรายวัน รายเดือน รายไตรมาส และรายปีตาม endpoint ที่บริการกำหนด

ข้อมูลชุดนี้เหมาะกับงานที่ต้องมีอัตราอ้างอิงสม่ำเสมอ เช่น การบันทึกรายการเงินตราต่างประเทศ การทำรายงาน หรือการปรับมูลค่ายอดคงเหลือปลายงวด

อัตราอ้างอิงอาจไม่เท่ากับอัตราที่ทำรายการจริง

เรตจาก ธปท. ไม่จำเป็นต้องตรงกับเรตที่ธนาคารหรือคู่ค้าใช้ชำระเงินจริง บริษัทจึงควรกำหนดนโยบายให้ชัดว่าจะใช้อัตราประเภทใดกับการบันทึกเอกสาร การรับ–จ่ายเงิน และการปรับปรุงปลายงวด

บริการข้อมูลวันหยุดธนาคาร

Financial Institutions’ Holidays API ให้ข้อมูลวันหยุดตามประเพณีและวันหยุดทำการเพิ่มเติมของสถาบันการเงิน ข้อมูลนี้นำไปใช้กับงานที่ขึ้นอยู่กับวันทำการของธนาคารได้ เช่น

  • วางแผนรับ–จ่ายเงิน — ตรวจสอบว่าวันครบกำหนดตรงกับวันหยุดของสถาบันการเงินหรือไม่
  • เตรียมไฟล์จ่ายเงิน — เลื่อนการอนุมัติและส่งไฟล์ให้ทันก่อนวันหยุด
  • ประมาณการกระแสเงินสด — ปรับวันที่คาดว่าจะรับหรือจ่ายให้ใกล้กับวันทำการจริง
  • สร้างการแจ้งเตือน — เตือนทีมการเงินล่วงหน้าเมื่อมีวันหยุดต่อเนื่องหรือวันหยุดเพิ่มเติม
วันหยุดธนาคารไม่ใช่ปฏิทินกลางของทุกงาน

ข้อมูลชุดนี้เป็นวันหยุดของสถาบันการเงิน ไม่ใช่วันหยุดของบริษัท ลูกค้า คลังสินค้า หรือผู้ขนส่ง หากจะใช้กับการจัดส่งและการวางบิล ต้องนำปฏิทินของแต่ละฝ่ายมาประกอบด้วย

นำข้อมูลมาใช้กับ Business Central ได้อย่างไร

Business Central มีพื้นที่สำหรับเก็บอัตราแลกเปลี่ยนและมี Base Calendar สำหรับกำหนดวันทำงานอยู่แล้ว การเชื่อม BOT API จึงช่วยลดขั้นตอนนำข้อมูลจากภายนอกเข้าระบบได้

ตัวอย่างการทำงาน

  1. ระบบเรียก BOT API ตามรอบเวลาและตรวจสอบผลการเชื่อมต่อ
  2. แปลงข้อมูลอัตราแลกเปลี่ยนเข้าสู่ Currency Exchange Rates ตามชนิดเรตที่บริษัทเลือก
  3. นำวันหยุดสถาบันการเงินไปสร้างหรืออัปเดตปฏิทินที่ใช้กับกระบวนการรับ–จ่ายเงิน
  4. เก็บวันที่อ้างอิงและประวัติการทำงาน เพื่อให้ตรวจสอบย้อนหลังและดึงข้อมูลซ้ำได้

สำหรับอัตราแลกเปลี่ยน BC รองรับการอัปเดตผ่านบริการภายนอก แต่ถ้ารูปแบบข้อมูลไม่ตรงกับกระบวนการนำเข้ามาตรฐาน อาจต้องเพิ่ม codeunit เพื่อแปลงข้อมูล ส่วน Base Calendar มาตรฐานเหมาะกับการคำนวณวันในงานขาย ซื้อ โอนย้าย และการผลิต งานรับ–จ่ายเงินหรือวันวางบิลที่มีกติกาเฉพาะต้องออกแบบเพิ่มให้ตรงกับองค์กร

สิ่งที่ต้องกำหนดก่อนเชื่อมต่อ

การเชื่อม API เป็นเพียงส่วนรับข้อมูล สิ่งสำคัญกว่าคือกติกาที่ใช้ตัดสินใจหลังได้รับข้อมูลแล้ว

  • ชนิดอัตราแลกเปลี่ยน — ระบุว่าแต่ละกระบวนการใช้อัตราซื้อ อัตราขาย อัตราเฉลี่ย หรืออัตราอ้างอิงใด
  • วันที่มีผล — กำหนดว่าจะใช้อัตราของวันเอกสาร วันลงบัญชี หรือวันทำการก่อนหน้า
  • วิธีจัดการวันหยุด — เลื่อนก่อนหรือหลังวันหยุด และใช้ปฏิทินของฝ่ายใดเป็นหลัก
  • กรณี API ไม่ตอบกลับ — แจ้งเตือน หยุดการอัปเดต หรือใช้ข้อมูลล่าสุดได้ภายใต้เงื่อนไขใด
  • การตรวจสอบย้อนหลัง — เก็บแหล่งที่มา วันที่อ้างอิง เวลาเรียกข้อมูล และผลการทำงาน

JWIC Localization ช่วยตรงไหน

ส่วนเสริมที่เราพัฒนารองรับการเชื่อม BOT API เพื่อดึงอัตราแลกเปลี่ยนและวันหยุดของสถาบันการเงินเข้า Business Central พร้อมบันทึกวันที่อ้างอิง ประวัติการทำงาน และแจ้งเตือนเมื่อเชื่อมต่อไม่สำเร็จ

ก่อนเปิดใช้งาน เราจะกำหนดชนิดเรต ปฏิทิน และกติกาการเลื่อนวันร่วมกับผู้ใช้งาน เพื่อให้ข้อมูลจาก ธปท. ถูกนำไปใช้ตรงกับนโยบายบัญชีและขั้นตอนทำงานของบริษัท

ศึกษารายละเอียดได้จาก บริการข้อมูลอัตราแลกเปลี่ยน และ บริการข้อมูลวันหยุดของสถาบันการเงิน บน BOT API Portal รวมถึงแนวทาง การอัปเดตอัตราแลกเปลี่ยน และ การตั้งค่า Base Calendar ใน Business Central

สรุป

BOT API ทำให้ระบบเรียกข้อมูลจาก ธปท. ได้โดยตรง อัตราแลกเปลี่ยนช่วยให้งานที่เกี่ยวกับเงินตราต่างประเทศใช้แหล่งอ้างอิงเดียวกัน ส่วนวันหยุดของสถาบันการเงินช่วยวางแผนงานรับ–จ่ายเงินได้ดีขึ้น ประโยชน์จะเกิดขึ้นจริงเมื่อกำหนดชนิดข้อมูล วันที่อ้างอิง และกติกาการใช้งานให้ชัดเจนก่อนเชื่อมต่อ
